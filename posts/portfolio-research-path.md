# Portfolio-Driven Research Path

The strategy is to make work visible enough that a serious reviewer can evaluate it without a resume parser.

## Operating Loop

1. Implement the paper or idea.
2. Publish notes and failure modes.
3. Write a concise technical summary.
4. Ask for targeted feedback.
5. Improve the artifact.

```text
implementation -> notes -> public artifact -> direct outreach -> feedback
```

| Channel | Purpose |
| --- | --- |
| GitHub | proof of implementation |
| Technical notes | proof of understanding |
| Direct email | focused outreach |
| Referrals | trust transfer |
